# AI、App、轻社交低成本方案

## 总原则

目标是尽量低成本、低认证、可维护。

优先顺序：

1. 保持网页版稳定。
2. Android APK 可分享安装。
3. AI 使用用户自填 API Key 或免费额度。
4. 轻社交先做好友码，不做复杂社交平台。

## App 打包

### Android

可行度：高。

不上架应用商店时：

- 不需要 Google Play 开发者账号。
- 不需要应用商店审核。
- 可以生成 APK 文件发给朋友。
- 朋友安装时需要允许“安装未知来源应用”。

建议技术路线：

- 先把网站做成 PWA 友好。
- 使用 Capacitor 打包 Android。
- 生成签名 APK。
- 保存签名密钥，后续更新必须使用同一密钥。

注意：

- 不要把签名密钥提交到公开仓库。
- APK 文件可以单独分享。
- 如果未来要上架，再准备 Google Play Console。

### iOS

可行度：低到中。

不上架也给朋友安装会比较麻烦，通常涉及：

- macOS
- Xcode
- Apple ID
- 设备限制
- 证书或 TestFlight

第一阶段不建议把 iOS 私发作为目标。iPhone 用户可先使用网页版或添加到主屏幕。

## AI 功能

### 推荐低成本方案

第一阶段建议做“用户自填 API Key”：

- 用户自己选择服务商。
- 用户自己填 Key。
- Key 只保存在用户本地浏览器。
- 网站不承担公共费用。

可支持服务商：

- DeepSeek
- 千问 / 阿里百炼
- Kimi
- 其他兼容 OpenAI 格式的接口

### 免费方案现实判断

完全免费、稳定、多人可用、不限量的云端 AI 基本不存在。

可考虑：

- 新用户免费额度。
- 用户自填 Key。
- 本地浏览器 AI，例如 WebLLM。

本地浏览器 AI 隐私最好，但手机性能、模型大小、浏览器兼容性会影响体验。

### AI 可做功能

- 陪伴聊天。
- 解释今天推荐这个任务的原因。
- 根据用户状态推荐低能量任务。
- 生成鼓励语。
- 帮用户创建自定义模式。
- 帮用户拆分大目标。
- 每周总结。
- 日记总结。
- 网站使用帮助。

### AI 隐私规则

- 默认不发送日记和打卡记录。
- 用户明确同意后才把数据发给 AI。
- 页面上清楚说明会发送哪些内容。
- 提供关闭 AI 的入口。
- 提供清空 AI 设置的入口。
- 不做医疗、心理诊断承诺。

## 轻社交功能

目标是类似页游好友系统，而不是大型社交平台。

### 第一阶段

- 本地昵称。
- 好友码。
- 添加好友。
- 好友列表。
- 今日打卡状态。
- 互相点赞或鼓励。

### 第二阶段

- 简单聊天。
- 小队打卡。
- 连续打卡榜。
- 每日一句互相鼓励。
- 隐私可见范围。

### 技术路线

低成本建议：

- Firebase
- Supabase

优先使用：

- 匿名账号
- 邮箱登录
- 好友码

暂不建议一开始接入：

- 微信登录
- QQ 登录
- 手机号短信登录

原因：

- 审核复杂。
- 可能需要资质。
- 短信通常要钱。
- 社交登录要遵守对应平台规则。

## 轻社交安全与合规

即使是小社交，也建议准备：

- 举报
- 拉黑
- 删除好友
- 隐私设置
- 昵称限制
- 基础敏感词过滤
- 数据删除能力

不要做：

- 抓取微信/QQ 好友。
- 模仿第三方登录。
- 使用非官方接口。
- 默认公开用户打卡日记。

## 成本压缩路线

最低成本路线：

1. 继续维护 GitHub Pages 网页版。
2. 新实验库做 2.0.0。
3. Android APK 使用本地打包，不上架。
4. AI 使用用户自填 API Key。
5. 轻社交使用 Firebase/Supabase 免费额度。
6. 不接短信，不接复杂社交登录。

这样可以把成本压到最低，同时保留未来升级空间。
