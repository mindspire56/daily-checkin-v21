# ROADMAP：2.0.0 实验方向

## 总目标

在 `1.4.4.2` 的基础上，开启一个更自由、更可扩展的 `2.0.0` 实验版本。

2.0.0 的重点不是简单堆功能，而是把每日打卡网站升级成：

- 可自定义的个人习惯空间
- 可打包成 Android App 的移动端工具
- 可选 AI 陪伴和任务建议
- 可选轻社交好友系统

## 仓库策略

不要直接在正式仓库 `mindspire56/daily-checkin` 上大改 2.0.0。

推荐创建实验仓库：

- `mindspire56/daily-checkin-v2`
- 或 `mindspire56/daily-checkin-lab`

实验站部署成功后，再提供独立预览链接：

- `https://mindspire56.github.io/daily-checkin-v2/`
- 或 `https://mindspire56.github.io/daily-checkin-lab/`

正式站继续保留当前 1.x 版本，保证可用、可回退。

## 2.0.0 可探索功能

### 页面架构

- 从单文件增强版逐步转向更清晰的结构。
- 可以继续保留静态部署能力。
- 如果要做 App、AI、社交，建议拆分为：
  - 页面组件
  - 数据管理
  - 本地存储
  - 可选云同步
  - 可选 AI 服务

### AI 陪伴

- 添加一个可开关的 AI 陪伴入口。
- 优先使用用户自填 API Key，不由站长承担公共调用费用。
- AI 功能必须明确提示是否会把日记、打卡记录发给模型。
- 默认不上传用户数据。
- 可做功能：
  - 陪伴聊天
  - 今日任务解释
  - 推荐更温和的任务
  - 根据用户输入生成鼓励语
  - 帮用户创建自定义模式
  - 网站使用帮助

### App 打包

- 优先 Android APK。
- 不上架应用商店时，不需要 Google Play 开发者账号。
- 需要生成签名密钥，后续更新必须保留同一个密钥。
- iOS 私发安装难度高，不建议作为第一阶段目标。
- 可先做 PWA，再用 Capacitor 打包 Android。

### 轻社交

目标是类似页游好友系统，不做复杂社交平台。

可做功能：

- 昵称
- 头像
- 好友码
- 添加好友
- 好友列表
- 简单聊天
- 今日打卡状态
- 互相鼓励
- 小队打卡

低成本建议：

- Firebase 或 Supabase。
- 优先匿名账号、邮箱登录或好友码。
- 暂不接微信、QQ 登录，避免审核、资质和费用复杂化。

## 2.0.0 第一阶段建议

1. 新建实验仓库。
2. 复制当前 1.4.4.2 页面作为起点。
3. 加入 `handoff` 文件夹。
4. 建立实验版本号 `2.0.0-alpha.1`。
5. 先做 App/PWA 友好的结构整理。
6. 再做 AI 设置入口和用户自填 API Key。
7. 最后再试轻社交。

## 不建议一开始做的事

- 不要直接覆盖正式站。
- 不要一开始接入站长自己的付费 AI Key 给所有人用。
- 不要先做复杂社交广场。
- 不要把所有功能同时塞进首页。
- 不要牺牲当前打卡主操作的简洁感。
