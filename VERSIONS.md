# 版本记录

## 2.1.0.5
- 紧急修复小橘猫面部模型 bug。
- 修正虎斑层级压住眼睛的问题：眼睛提升到最前层，额头虎斑上移并退到后层。
- 缩小并重定位鼻口区域，避免面部中心比例异常。
- 轻微弱化胡须和虎斑透明度，让小猫面部更清晰。
- 快照：`versions/2.1.0.5/index.html`

## 2.1.0.4
- 参考用户提供的小橘猫图片，将伙伴模型重建为单一小橘猫风格。
- 小橘猫模型新增蓝灰大眼、粉色鼻子、白胸口、白爪、橘色虎斑、胡须和右侧尾巴等细节。
- 移除领养小狗选项，伙伴系统改为只保留小橘猫。
- 旧版已领养小狗的数据会自动迁移为小橘猫，避免历史数据显示异常。
- 保留免费互动、投喂和亲密度体系。
- 快照：`versions/2.1.0.4/index.html`

## 2.1.0.3
- 将伙伴页升级为无外部依赖的 CSS 3D 伙伴模型，保留小猫和小狗区分。
- 新增 3D 透视层次：身体、头部、耳朵、眼睛、尾巴分层组合，呈现立体感。
- 新增免费互动：点击伙伴可摸摸，按钮支持摸摸、招手、转圈，不消耗积分。
- 保留投喂和亲密度体系，投喂仍作为成长反馈，不再是唯一互动方式。
- 快照：`versions/2.1.0.3/index.html`

## 2.1.0.2
- 将“我的伙伴”从静态 CSS 形象升级为轻动态伙伴。
- 小猫和小狗新增待机呼吸、眨眼、尾巴轻晃动画。
- 投喂当天伙伴进入更活泼的开心状态；夜间自动进入睡觉状态。
- 动画继续使用轻量 CSS 实现，不引入外部模型库，保持手机端加载轻快。
- 快照：`versions/2.1.0.2/index.html`

## 2.1.0.1
- 新增第一版“我的伙伴”陪伴页，入口放在鼓励语下方，避免抢占首页打卡主线。
- 支持首次从小猫和小狗中领养一位伙伴，并可给伙伴取名。
- 新增积分投喂：使用所有积分扣除已投喂消耗后的可用点数，兑换小鱼干/小骨头、温牛奶、能量饼干、今日小餐。
- 新增伙伴亲密度：投喂会增加亲密度，领养后每日首次完成打卡也会增加亲密度。
- 伙伴状态采用温柔反馈，不设置饥饿惩罚、亲密度下降或负面催促。
- 指引系统总览新增“我的伙伴”入口说明。
- 快照：`versions/2.1.0.1/index.html`

## 2.1.0.0
- 新突破线起点版本，基于 2.0.0.3 的稳定可用基础创建。
- 2.0.0.3 作为可正常使用的稳定实验版封存，不继续在 `daily-checkin-v2` 仓库中做大突破。
- 新建独立实验库用于后续 AI 陪伴、App、轻社交、架构升级等大突破方向。
- 2.1.0.0 使用独立 localStorage 键 `daily-checkin-v21`，避免后续实验污染 2.0.0.3 的本地数据。
- 雨声音乐暂时引用 `daily-checkin-v2` 已稳定上传的资源，避免新库重复上传大音频文件。
- 快照：`versions/2.1.0.0/index.html`

- 更新音乐栏指引，覆盖默认曲库、本地音乐、播放模式和排序。
